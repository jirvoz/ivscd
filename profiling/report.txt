In the output of profiling of standard deviation, using functions
from mathematical library "CDMath" we found out, that program calls
the most following functions: "cfrac", "ln" and "fabs". These functions
are used by function "power" in order to get power of a nubmer.

Function "power" was called in total 						1002 times.

The program supposedly spend the most of time on function "cfrac"

That means that in optimalisation of the program and its code we should
consider focusing on function cfrac.

 README for school project of subject "Practical aspects of software design"

 IVSCD Calculator can evaluate expressions with operators **+-*/^**, **brackets** and few basic **functions** like **sin**, **cos**, **tan**, and **natural and common logarithm**.

## Build
Building main application:
```
$ cd src    # cd to source code
$ qmake     # generate makefile
$ make      # compile main application
```
Tests:
```
$ cd src       # cd to source
$ qmake        # generate makefile
$ make tests   # build and run tests
```

## Install
Debian package:
```
$ cd src       # cd to source
$ qmake        # generate makefile
$ make deb     # create debian package
$ sudo dpkg -i ivs_1.0-1.deb	 # install with dpkg
```
Direct installation:
```
$ cd src        # cd to source
$ qmake         # generate makefile
$ make install  # install app (make unistall to remove)
```
